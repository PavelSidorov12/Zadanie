git init - Создаем новый "дневник" для проекта, чтобы начать записывать все изменения.
git clone <ссылка> - Копируем чужой "дневник" проекта на свой компьютер, чтобы работать с ним.
git add <файлы> - Говорим Git, что хотим сохранить последние изменения в этих файлах.
git commit -m "Сообщение" - Сохраняем запись об изменениях с кратким описанием того, что сделали.
git status - Проверяем, какие изменения мы сделали, но ещё не сохранили.
git branch <имя> - Создаем новую "страницу" в "дневнике", чтобы работать над чем-то отдельно.
git checkout <имя> - Переключаемся на другую "страницу" или "раздел" в нашем "дневнике".
git merge <имя> - Складываем вместе изменения из одной "страницы" в другую, чтобы все было вместе.
git pull - Забираем последние записи из "дневника" в интернете, чтобы наш копия была актуальной.
git push - Отправляем наши записи в "дневник" в интернете, чтобы другие могли их видеть.
git fetch - Только смотрим последние изменения из интернета, но не добавляем их к себе.
git log - Читаем историю всех записей, которые мы или кто-то другой сделал.
git diff - Смотрим разницу между текущими изменениями и последними сохраненными.
git stash - Временно прячем изменения, чтобы рабочее место было чистым, и можно было быстро переключиться на другую задачу.
git reset <файл/коммит> - Отменяем изменения в файлах или возвращаемся назад в истории, если что-то пошло не так.
